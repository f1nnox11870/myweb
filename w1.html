What is Data Fabric

Data Fabric is a modern data design concept to better integrate and connect processes across resources and endpoints. By using both human and machine capabilities, the Data Fabric identifies and connects disparate data. This supports faster decision-making. Optimized engineering restructuring and improved data management practices. For example, you might think of a data fabric as a passive data observer that only acts when it finds an asset that needs to be managed. Based on specific applications, Data Fabrics can automatically manipulate data and provide recommendations for data choices.

How does it work?

LoTo better understand how the Data Fabric works, let's take a look at 6 different Data Fabric layers: Data management — This layer is responsible for data security and governance. Data Import — This layer searches for connections between structured and unstructured. Data processing — This layer refines the data for accurate and relevant retrieval of data. Data Synchronization — This layer makes data available to teams by transforming. Merge and Cleanup Data discovery — This layer opens up new opportunities to integrate and develop insights from different data sources. Data access — Finally, this layer ensures that permissions and conditions are compliant and allows access through virtual dashboards. This integrated and layered approach to data management protects organizations from the most prevalent types of attacks, such as client-side, supply chain, business apps, and even automated attacks.
